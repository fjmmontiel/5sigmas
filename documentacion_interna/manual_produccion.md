# Manual de Producción de Contenido - 5sigmas

Este documento unifica los blueprints operativos para texto, vídeo y podcast. Su objetivo es maximizar la retención y mantener una única línea editorial coherente.

---

## 1. Blueprint Editorial Común (Transversal)

**Obligatorio para cualquier formato.**

### 1.1. Tono y Propósito

*   **Propósito**: Separar hype de conocimiento sólido. Conectar fundamentos (filosofía, matemáticas, ingeniería, energía) con producto, proyecto y estrategia.
*   **Tono**:
    *   Directo, sobrio, sin exageraciones.
    *   Anti-hype, pro-evidencia.
    *   Didáctico pero no paternalista.
*   **Voz**:
    *   Hablar como “arquitecto/estratega” que ha bajado cosas a producción.
    *   Evitar claims que no puedas justificar con datos, papers o experiencia.

### 1.2. Estructura Narrativa Base

Usar siempre una variante de **Hook → Problema → Fundamento → Aplicación → Cierre/CTA**.

1.  **Hook** (5–20s / 1–3 frases): Captar la atención inmediatamente.
2.  **Problema**: Definir qué le duele al decisor.
3.  **Fundamento**: Concepto teórico explicado con claridad.
4.  **Aplicación**: Casos de uso, decisiones, trade-offs.
5.  **Cierre**: 3 bullets clave + CTA suave.

## 2. Blueprint para TEXTO (Post/Blog/Newsletter)

### 2.1. Estructura

1.  **Título**: Claro, sin clickbait barato. (Ej: "Concepto → Para quién → Resultado").
2.  **Hook (1–3 frases)**: Situación concreta (“Eres Head of Product y…”) o pregunta fuerte.
3.  **Problema (4–8 frases)**: Nombra el problema (ruido, coste, riesgo) y agítalo (consecuencias).
4.  **Fundamento (Bloque Central)**:
    *   2–4 subsecciones (una idea por subsección).
    *   Si hay fórmula/diagrama, explicarlo intuitivamente.
5.  **Aplicación a Negocio/Estrategia**:
    *   Implicaciones para C-level, Product, Tech Lead.
    *   Cambios en Roadmap, Arquitectura, KPIs.
6.  **Takeaways + CTA**:
    *   3 bullets resumen.
    *   CTA a siguiente post, podcast o vídeo.

## 3. Blueprint para VÍDEO (YouTube / Shorts largos)

### 3.1. Estructura por minuto (8–15 min)

1.  **0:00–0:10 – Hook**: Declaración directa del problema. Sin intros largas.
2.  **0:10–0:30 – Promesa + Mapa**: Qué vas a entender y qué tendrás al final (checklist).
3.  **0:30–2:00 – Problema + Contexto**: Error típico o ejemplo real.
4.  **2:00–8:00 – Fundamento + Aplicación**:
    *   3 bloques de 2-3 min.
    *   Concepto base + Error típico + Cómo se hace bien.
    *   Cambio visual cada 60-90s.
5.  **8:00–10:00 – Casos/Práctica**: 1-2 ejemplos (Contexto → Decisión mala → Decisión buena → Resultado).
6.  **10:00–11:00 – Cierre + CTA**: 3 bullets en pantalla + 1 CTA claro.

### 3.2. Instrucciones Técnicas

*   **Título/Thumbnail**: Alineados con el problema real.
*   **Ritmo**: Cambio visual cada 5–10s. Esquemas minimalistas.
*   **Reutilización**: Sacar 2–4 cortes de 30–60s (Shorts) de cada vídeo largo.

## 4. Blueprint para PODCAST

### 4.1. Estructura de Episodio (20–40 min)

1.  **Intro (0:00–1:00)**: Hook fuerte + Para quién es + Qué decidirá mejor.
2.  **Segmento 1 – Problema y Contexto (5–8 min)**: Descripción concreta de situaciones (comité, producto). Agitar costes/riesgos.
3.  **Segmento 2 – Fundamento (8–15 min)**: 1-2 conceptos clave. Nivel pizarra (sin slides). Máximo 3 conceptos.
4.  **Segmento 3 – Aplicación (10–20 min)**: Casos prácticos y Checklist hablado ("5 preguntas antes de...").
5.  **Cierre (2–3 min)**: Recap en 3 bullets + CTA (compartir).

### 4.2. Producción

*   **Intro fija**: 5–10s máx.
*   **Sonido**: Prioridad absoluta a la claridad de voz.
*   **Guion**: Usar escaleta, no lectura literal.

---

## 5. Resumen Operativo (El Filtro)

Toda pieza debe responder:
1.  ¿Qué problema real del decisor ataco?
2.  ¿Qué fundamento explico?
3.  ¿Qué decisión puede tomar mejor después?

Tono: **Anti-hype, pro-estructura.** Siempre sacar a la luz los trade-offs (coste, riesgo, energía, tiempo).
